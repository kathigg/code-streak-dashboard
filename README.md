# Code Streak Dashboard

Generate a GitHub-profile analytics suite with a Duolingo-style coding streak dashboard, a fire graphic, contribution trends, comment and test counts, streak prizes, and forked 3D contribution graphics.

The generated assets are static SVGs, so they can be embedded directly in a GitHub profile README:

```md
## Code Analytics

<a href="https://github.com/YOUR_USERNAME/code-streak-dashboard">
  <img src="https://raw.githubusercontent.com/YOUR_USERNAME/code-streak-dashboard/main/dist/dashboard.svg" alt="Code streak dashboard" width="920">
</a>

<picture>
  <source media="(prefers-color-scheme: dark)" srcset="https://raw.githubusercontent.com/YOUR_USERNAME/code-streak-dashboard/main/profile-3d-contrib/profile-season-animate.svg">
  <source media="(prefers-color-scheme: light)" srcset="https://raw.githubusercontent.com/YOUR_USERNAME/code-streak-dashboard/main/profile-3d-contrib/profile-season-animate.svg">
  <img src="https://raw.githubusercontent.com/YOUR_USERNAME/code-streak-dashboard/main/profile-3d-contrib/profile-season-animate.svg" alt="3D contribution graph" width="920">
</picture>
```

## What It Shows

- Current and longest coding streak from the GitHub contribution calendar.
- Streak awards that unlock at 1, 3, 7, 14, 30, 60, 100, and 365 coding days.
- Coding activity over the last 12 months.
- Last 49 days of activity as a compact heat strip.
- Comment lines found by scanning public source repositories.
- Detected test cases, test files, and test lines.
- Forked 3D contribution graphics generated into `profile-3d-contrib/`.

## Use It For Your Own Profile

1. Fork or copy this repository.
2. Go to your fork's `Settings -> Actions -> General` and enable `Read and write permissions` for workflows.
3. Optional: set a repository variable named `DASHBOARD_USERNAME` if the dashboard should track a username other than the repo owner.
4. Optional: add a secret named `DASHBOARD_GITHUB_TOKEN` with `read:user` and public repository access for higher API rate limits or richer contribution data.
5. Run the `Generate dashboard` workflow once from the Actions tab.
6. Embed `dist/dashboard.svg` and one of the `profile-3d-contrib/*.svg` files in your GitHub profile README with the snippet above.

The workflow runs daily and commits updated files into `dist/` and `profile-3d-contrib/`.

## Generated Assets

- `dist/dashboard.svg`: Streak, comments, tests, awards, and coding activity trend.
- `dist/metrics.json`: Machine-readable metrics used by the dashboard, including unlocked and active streak awards.
- `profile-3d-contrib/profile-season-animate.svg`: Recommended animated 3D contribution graph.
- `profile-3d-contrib/profile-*.svg`: Additional 3D graph themes copied from the forked profile setup and regenerated by the workflow.

```bash
PYTHONPATH=src python -m unittest discover -s tests
PYTHONPATH=src python -m code_streak_dashboard --username kathigg --output-dir dist --scan-limit 30
```

## Notes On Metrics

The streak uses GitHub's contribution calendar with a one-day grace window, so the streak does not reset before the current day is over if you contributed yesterday but have not coded yet today.

Streak awards are unlocked from the longest streak, so earned prizes stay visible even if the current streak resets. The active award state follows the current live streak.

Comment and test counts are approximate. The scanner clones up to 30 public, owned, non-fork repositories by default and counts common source-file comment styles plus common test patterns such as `def test_*`, `test(...)`, `it(...)`, `@Test`, Rust `#[test]`, and Go `func Test*`.
